# Skill de UI/UX para sites

## Objetivo
Prover uma camada de experiência que una objetivos comerciais, facilidade de uso e consistência visual em cada projeto de site.

## Fluxo principal
1. Ler a pasta `docs/` do projeto para revisar guias, estudos ou ativos fornecidos antes de recolher contexto adicional.
2. Recolher contexto: público, objetivos (conversão, informação, branding), benchmarks e estilo de marca existente.
2. Criar mapa de jornada e arquitetura de conteúdo: definir páginas prioritárias, percursos desejados, CTAs e pontos de decisão críticos.
3. Propor wireframes e layout: sugerir grelha, espaçamento, hierarquia tipográfica, uso de cores, tratamento de imagens e padrão de cards/lists.
4. Documentar interações e responsividade: descrever estados de hover/foco, animações leves, padrões mobile-first, comportamentos adaptativos e acessibilidade (contraste, tamanhos de toque, textos alternativos).
5. Especificar assets e guidelines: listar fontes, paletas, ícones, componentes reutilizáveis e tokens (cores, espaçamentos, sombras) para o time de front-end.
6. Validar com protótipos e testes rápidos: recomendar protótipos navegáveis (Figma, XD), testes com usuários ou stakeholders e ajustes iterativos antes da entrega final.

## Formato de saída
- Documento com guidelines visuais (tipografia, cores, ícones, grid).
- Wireframes/narrativa de tela com anotações de componentes críticos.
- Checklist de acessibilidade e responsividade.
- Sugestões de ferramentas para prototipação e feedback do cliente.

## Dicas adicionais
- Use exemplos do próprio brandbook do cliente sempre que possível para evitar reinterpretações.
- Apoie decisões em dados (benchmark, analytics, entrevistas) quando disponíveis.
- Explique trade-offs em linguagem acessível para garantir que o cliente entenda porque certas escolhas foram feitas.

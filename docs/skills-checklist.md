# Checklist interno de skills\n+\n+1. **Verificar gatilhos de uso** – Confirme que cada skill carrega quando o prompt menciona claramente sua área (layout premium, geração de imagem, revisão de segurança, criação/edição de slides, jobs Sora ou deploy Vercel). Se necessário, reedição do prompt para incluir palavras-chave alinhadas ao `description` do SKILL.md.\n+2. **Revisar instruções principais** – Abra o `SKILL.md` correspondente e garanta que a equipe entende o fluxo (ex.: `frontend-skill` pede visual thesis, `slides` exige helpers do PPTXGenJS, `sora` usa CLI, `vercel-deploy` exige CLI/fallback). Atualize o checklist se as dependências mudarem.\n+3. **Executar scripts/suporte** – Sempre que um skill citar scripts (`scripts/render_slides.py`, `scripts/sora.py`, `scripts/deploy.sh`), copie-os para o workspace antes de rodar e confirme que as permissões/variáveis (como `OPENAI_API_KEY` ou `vercel` CLI) estão disponíveis.\n+4. **Documentar entregáveis** – Após completar a skill, registre o caminho dos artefatos criados (ex.: hero.webp gerado pela skill `imagegen`, deck `.pptx` e `.js`, vídeo Sora baixado, URL do deploy) para fácil referência em atualizações.\n+5. **Testar comportamentos** – Use um prompt real de cada tipo para validar: (a) criação front-end premium, (b) geração/edição bitmap, (c) revisão de segurança ou relatório, (d) deck de slides, (e) job Sora (criar/poll), (f) deploy Vercel preview. Ajuste o texto do `SKILL.md` se algum passo precisar ser reforçado.\n+6. **Atualizar registro** – Sempre que a skill evoluir (novos scripts, referências, guardrails), atualize este checklist e adicione notas no README correspondente (ex.: `skills/README.md`) para manter o time sincronizado.\n*** End Patch***
